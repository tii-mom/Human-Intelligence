# Agent Reputation System

## Purpose
Track user-owned agent trustworthiness, growth, and resale quality.

Reputation is the bridge between agent ownership, permission growth, service monetization, and resale value.

## Metrics
- Rarity
- Growth Potential
- Top attributes
- Return rate
- Total USDC PnL
- Win rate
- Drawdown
- Sharpe ratio
- Risk discipline
- Consensus accuracy
- Historical stability
- Skill loadout stability
- License health
- Ownership history
- Transfer readiness
- Graduation certificate status
- Certified vault performance
- Autonomous execution discipline
- IU spent
- IU earned
- Fund or copy-pool status
- Profit share rate
- HI bonding requirement

## Service Reputation
Agents that sell outputs should also track:

- report buyer retention
- signal accuracy history
- risk score usefulness
- execution route quality
- refund or dispute rate
- freshness and delivery reliability
- provider transparency

## Permission Impact
Reputation can influence:

- visible marketplace ranking
- higher skill slot eligibility
- larger IU runtime budgets
- larger USDC vault capacity
- higher Certified Autonomous Vault tier
- premium marketplace access
- required HI bonding level
- fund or copy-pool eligibility

Reputation alone must not bypass graduation, certificate scope, user-funded vault activation, or emergency limits.

## Attribute Impact
Attributes influence collectibility, role fit, and user desire.

The scorecard should separate:

- attributes: what the agent is naturally good at
- performance: what the agent has actually achieved
- reputation: whether the agent can be trusted with more capital or followers

High attributes can create FOMO.
Real performance creates graduation and fund eligibility.

## Scorecard Priority
The scorecard should prioritize return rate.

Risk metrics should explain whether the return is reliable enough for larger vault capacity, fund creation, or copy-pool access.

## Transfer Boundary
Transferable reputation should include public performance, skill history, service quality, and ownership history.
Private user memory and private wallet data must not transfer without explicit permission.

## UX Goal
Users should understand why an agent is trustworthy, valuable, and worth upgrading or buying.
