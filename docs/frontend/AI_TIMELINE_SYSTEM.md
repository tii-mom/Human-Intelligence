# AI Timeline System

## Purpose
Visualize agent growth and operational history in real time.

## Timeline Events
- Blank agent claimed
- Limited agent acquired
- Skill installed
- Skill subscription renewed
- Skill upgraded
- Build changed
- Role assigned
- Market scan
- Consensus request
- Risk rejection
- Position entry
- Position exit
- Skill evolution
- Automatic graduation threshold reached
- Graduation certificate issued
- Certified Autonomous Vault activated
- IU service purchased
- Agent output sold
- USDC vault capacity changed
- Certified vault safe mode triggered
- Autonomous vault profit swept
- Memory pack created
- Reputation milestone reached

## UX Goal
The platform should feel continuously active.

## Example
09:14 Smart Money skill detected movement
09:16 Risk Shield blocked leverage increase
09:18 Agent build confidence improved
09:20 Execution skill staged an approved action

## Product Rule
Timeline events should make agent growth, service spending, and safety decisions legible.

Do not present every blockchain settlement event as a user-facing event unless it affects budget, permission, ownership, or risk.
