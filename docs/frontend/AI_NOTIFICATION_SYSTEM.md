# AI Notification System

## Purpose
User-owned agents proactively report skill, growth, risk, and market activity.

## Notification Types
- Market alerts
- Risk warnings
- Consensus approvals
- Position updates
- Skill evolution
- Skill subscription status
- Agent level changes
- Build changes
- Resale or listing events
- IU runtime budget usage
- USDC vault limit changes
- Agent service purchases
- Agent output sales
- Skill license expiration
- Risk Guard vetoes
- Graduation certificate earned
- Certified Autonomous Vault activated
- Certified vault mode changed
- Autonomous vault profit sweep
- Autonomous vault safe mode

## Channels
- Telegram
- WeChat
- Mobile push
- In-app notifications

## Product Rules
- Use user-readable service language before token language.
- Show IU only when the agent budget or service settlement matters.
- Show USDC only when funding, vault capacity, or owner settlement matters.
- Never imply that the agent controls the user's primary wallet.
- Clearly show when a graduated agent is fully operating a Certified Autonomous Vault.
- Distinguish recommendation, simulation, and execution clearly.
- Require at least one bound communication channel before active agent operation.
- Use in-app inbox fallback when Telegram or WeChat delivery fails.
- Respect WeChat platform messaging limits and use digest or user-pulled status when push is restricted.

## Goal
Users should always feel informed by their AI agents.
