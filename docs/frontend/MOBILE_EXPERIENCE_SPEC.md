# Mobile Experience Specification

## Philosophy
HI Protocol is mobile-first and chat-native for managing user-owned agents.

Telegram is the first global channel.

WeChat is the priority channel for Chinese users.

## Requirements
- One-hand interaction
- Smooth animations
- Floating bottom navigation
- Responsive layouts
- Fast loading
- GPU optimized motion
- Agent builder access
- Skill loadout management
- Growth and risk visibility
- Telegram or WeChat binding after agent creation
- Chat command access

## Primary Mobile Flows
- claim free blank agent
- name the agent
- bind Telegram or WeChat
- receive first agent greeting
- chat with the active agent
- buy or view limited agents with HI
- install starter skills
- assign an agent role
- review agent growth
- review graduation certificate progress
- activate Certified Autonomous Vault
- approve skill or service purchases
- monitor IU activation and service budget
- monitor Compute Token balance
- monitor USDC vault capacity
- monitor autonomous vault PnL
- receive risk warnings
- review and sell agent outputs

## UX Boundary
Mobile should hide infrastructure by default.
It should present AI services, agent skills, budget impact, and risk status before showing payment or chain details.

## UX Goal
The mobile experience should feel like a premium control surface for growing an AI trading agent.
