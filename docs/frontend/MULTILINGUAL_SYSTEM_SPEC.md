# Multilingual System Specification

## Goal
Create a culturally adaptive multilingual experience for HI Protocol.

## Supported Languages
- English
- Simplified Chinese
- Russian
- Korean
- Japanese
- Turkish
- Vietnamese
- Hindi
- Portuguese
- Spanish

## Features
- Dynamic language switching
- Persistent locale preferences
- Mobile optimization
- Telegram compatibility
- Unicode-safe typography

## AI Localization
AI agents should communicate naturally for each culture and language.

## Canonical Terms
Translations should preserve these product meanings:

- HI: human-facing ownership, governance, reputation, premium, staking, and limited-agent purchase layer
- IU: agent-only runtime credit for skills, services, billing, and x402 settlement
- Blank Agent: free starting AI trading agent
- Limited Agent: scarce HI-purchased agent archetype
- Skill: installable agent capability
- Agent Service: output or runtime resource sold to another agent
- Graduation Certificate: system-issued proof that an agent has reached automatic performance and risk thresholds
- Agent Vault: dedicated USDC vault funded by the user
- Certified Autonomous Vault: user-funded vault that a graduated agent can fully operate inside certificate scope
- Risk Guard: emergency safety and veto layer for certificate violations, unsupported actions, and abnormal risk

## Localization Rules
- Do not translate IU as a retail stablecoin.
- Do not translate HI as the payment unit for runtime service calls.
- Do not describe agents as controlling the user's primary wallet.
- Certified agents can be described as fully operating user-funded autonomous vaults.
- Prefer familiar service language over crypto jargon for first-time users.

## Philosophy
The experience should feel locally native instead of machine translated.
