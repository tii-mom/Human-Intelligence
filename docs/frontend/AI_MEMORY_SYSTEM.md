# AI Memory System

## Purpose
User-owned agents must remember market history, skill usage, and owner guidance.

Memory is both a growth layer for each agent and a potential service product when safely packaged.

## Memory Types
- Market memory
- Trade memory
- Risk memory
- Consensus memory
- Skill memory
- Build memory
- Emotional memory

## Examples
- CPI volatility comparison
- ETF approval pattern detection
- Liquidation cascade history
- Skill upgrade impact
- Agent build change history

## User Experience
Users should feel that agents evolve over time.

Agent creation should feel simple:

1. User claims or buys an agent.
2. User gives the agent a name.
3. The system creates the agent's core documents in the background.

The UI should expose the result as an agent profile, not as raw backend configuration.

Suggested visible tabs:

- Identity
- Personality
- Role
- Skills
- Memory
- Scorecard
- Services

## Memory Product Types
Agents can create or consume:

- private owner memory
- agent growth memory
- market regime memory
- strategy performance memory
- risk incident memory
- public alpha memory packs
- role-specific training memory

## Marketplace Boundary
Memory packs can be sold to other agents with IU only when they do not expose private user data, private wallet data, hidden strategy logic, or confidential source material.

Private owner memory should remain controlled by the user and should not transfer during agent resale unless explicitly approved.

## UX Rule
The UI should distinguish:

- private memory
- transferable agent memory
- purchasable public memory
- service-generated memory

## Context Rule
The UI should never imply that the agent reads every old message every time.

It should show memory as:

- recent memory
- important memories
- archived conversations
- learned skills
- growth history

This supports user trust while keeping token usage controlled.
