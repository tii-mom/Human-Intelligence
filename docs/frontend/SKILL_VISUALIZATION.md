# Skill Visualization

## Purpose
Skills represent installable AI capabilities for user-owned agents.

## Core Features
- Rarity system
- Evolution tree
- Fusion history
- Crisis experience
- Market history
- License status
- Subscription and buyout pricing
- Compatible agent slots

## Service Fields
Skill cards should show:

- compatible agent roles
- earning path: USDC, IU, or both
- output type
- required data permissions
- IU cost model
- expected runtime usage
- expected USDC value contribution
- expected IU monetization path
- whether outputs can be resold
- whether the skill can request execution
- whether the skill can access private memory
- attribute effects
- Growth Potential compatibility

## Rarity
- Common
- Rare
- Epic
- Legendary

## Skill Card Rules
- Keep descriptions short
- Explain outcome, not mechanism
- Hide core strategy details
- Show earning path without promising profit
- Show risk level
- Show license type
- Show compatible builds

## License Display
Skill cards should distinguish:

- free starter
- subscription
- pay per use
- buyout
- resale eligible
- non-transferable

## Safety Display
Skills that touch execution, private memory, external data, or vault permissions need prominent safety labels.

## Graduation Impact
Skill cards should show whether the skill contributes to:

- automatic graduation score
- attribute growth
- return-rate improvement
- drawdown control
- IU service income
- Certified Autonomous Vault eligibility

## Attribute Effects
Skills can improve or activate attribute strengths:

- Alpha skills improve opportunity discovery
- Risk skills improve drawdown discipline
- Execution skills improve trading efficiency
- Research skills improve analysis quality
- Memory skills improve adaptation
- Personality and reporting skills improve Charisma

## UX Goal
Skills should feel valuable, useful, and safely installable without exposing proprietary logic.
