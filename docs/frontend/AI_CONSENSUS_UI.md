# AI Consensus UI

## Purpose
Multiple skills and agents collaborate before high-impact action.

Consensus should explain how an owned agent reasons, when it buys external agent services, and why a recommendation or execution request is allowed or blocked.

## Core Systems
- Voting engine
- Confidence scoring
- Conflict detection
- Consensus approval
- Risk veto
- Skill output comparison

## Visualization
- AI discussion flow
- Confidence bars
- Approval states
- Disagreement indicators
- Consensus wave animation

## Final Authority
Risk AI always has veto authority.

Certificate scope, vault mandate, and emergency limits remain above agent consensus.

For Certified Autonomous Vaults, consensus should support autonomous execution rather than require user approval for every trade.

## Agent Service Consensus
When an agent wants to buy an external output, the UI should show:

- service provider agent
- output type
- IU cost
- expected benefit
- confidence impact
- budget impact
- provider reputation
- risk or privacy implications

Examples:

- Portfolio agent requests a weekly report from a news agent.
- Execution agent requests a route from an execution service agent.
- Risk agent buys a volatility memory pack before approving leverage.

## Product Rule
Consensus should help users understand how their agent's installed skills formed an action.

Consensus must distinguish primary wallet safety from certified vault autonomy.
