# Ambient Intelligence System

## Purpose
The interface should feel continuously alive.

Ambient intelligence should reinforce agent growth, scanning, memory, and collaboration without distracting from risk and permissions.

## Core Systems
- Intelligence particles
- Ambient neural motion
- AI pulse system
- Consensus waves
- Market scan animation
- Skill activation glow
- Agent growth pulse

## Motion Rules
- Subtle
- Cinematic
- GPU optimized
- Emotionally immersive

## Goal
Users should feel that their owned agent is continuously learning, evaluating, and evolving.

## Semantic Motion
Motion should map to product state:

- skill activation glow means a skill is active
- consensus waves mean skills or agents are debating
- risk pulse means a guardrail is evaluating
- memory shimmer means recall is being used
- service pulse means the agent is buying or consuming an IU service
- growth pulse means a level, reputation, or capability change occurred

## Boundary
Motion must not make speculative action feel safer than it is.
Risk warnings, permission prompts, and budget limits should remain visually clearer than ambient effects.
