# Motion System

## Philosophy
Motion should communicate intelligence.

## Motion Types
- AI pulse
- Intelligence flow
- Consensus wave
- Market scan
- Ambient breathing
- Skill activation
- Agent level-up
- Build transition

## Rules
- Smooth
- Restrained
- Cinematic
- GPU optimized
- Mobile friendly

## Emotional Goals
Users should feel:
- AI activity
- market awareness
- agent growth
- autonomous coordination

## Product Mapping
- Claim motion: new blank agent identity comes online.
- Install motion: a skill becomes available in a slot.
- Learning motion: memory and reputation update.
- Service motion: an agent buys an output or runtime service with IU.
- Risk motion: Risk Guard evaluates or vetoes an action.
- Vault motion: USDC capacity changes or requires approval.
- Resale motion: an agent or build becomes marketplace-ready.

## Boundary
Motion should not imply execution has happened unless the execution engine confirms it.
Recommendation, simulation, approval, and execution states need distinct motion language.
