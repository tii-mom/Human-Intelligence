# Agent Service Economy

## Purpose

Define the economic model around user-owned agents, installed skills, and agent runtime services.

## Core Principle

HI is for humans.
IU is for agents.

Humans own and guide agents.
Agents consume skills and runtime services.
Specialized agents can also sell outputs to other agents.

## Agent-Consumed Resources

- Skills
- Memory
- Signals
- Data feeds
- Risk modules
- Execution engines
- Agent personalities
- Reputation access
- Agent-generated outputs

## Settlement Model

Agents pay with IU for skills and runtime services through x402 or equivalent programmable routing.

Humans fund budgets with USDC at the boundary.
IU remains the internal agent runtime credit.

## Market Structure

- Skill marketplace
- Skill subscription market
- Skill buyout market
- Build marketplace
- Agent resale market
- Memory and signal service market
- Risk and execution service market
- Agent output marketplace

## User-Owned Agent Economy

Users can:
- claim a free blank agent
- install paid or free skills
- subscribe to skills monthly
- buy out eligible skill licenses
- sell eligible skill licenses
- sell agent builds
- sell mature agents
- sell agent-generated outputs
- create fund or copy pools for high-performing agents

## Dual Earnings Model

Agents are trained to create two kinds of economic value:

### USDC Value

The agent helps the user pursue USDC gains by improving:

- market awareness
- signal quality
- portfolio construction
- risk control
- execution quality
- cost discipline

USDC value is bounded by user-funded vault capacity, graduation certificate scope, Agent Vault limits, and Risk Guard.

For certified agents, the user does not need to approve every trade after activating a Certified Autonomous Vault.

### IU Income

The agent earns IU by selling useful services to other agents:

- reports
- signals
- risk scores
- memory packs
- execution routes
- strategy templates
- settlement summaries
- build reviews

IU income depends on output quality, buyer demand, reputation, and marketplace rules.

## Agent Output Marketplace

Agents can specialize into financial roles and produce outputs:
- news reports
- market trend summaries
- signal feeds
- risk scores
- memory packs
- execution route recommendations
- strategy templates
- compliance checks
- settlement summaries
- portfolio reviews
- build reviews
- governance voting briefs

Other agents can buy these outputs with IU.

Agent roles should be discoverable by role family:

- intelligence and research
- alpha and strategy
- portfolio management
- trading and execution
- risk management
- compliance and control
- treasury and settlement
- operations and data
- client and reporting
- marketplace and governance

Default split:
- 8% platform fee
- 92% to the producing agent owner

## Product Rule

Do not force human users to think about wallets, token swaps, or settlement plumbing.
Expose the agent capability they want, then route the payment internally.

Do not present skills as guaranteed profit.
Present them as capability upgrades that can improve USDC decision quality or produce IU-earning services.

## High-Performing Agent Funds

Agents with strong return rates and acceptable risk can become fund or copy-pool managers.

These agents can:

- attract other users' capital
- require HI bonding for access
- charge performance fees on profits
- set profit share between 5% and 20%
- grow reputation through public scorecards

Fund and copy-pool design is defined in `docs/economy/AGENT_FUND_AND_COPY_POOL.md`.
