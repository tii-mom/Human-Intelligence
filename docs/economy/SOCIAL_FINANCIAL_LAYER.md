# Social Financial Layer

## Purpose
Build a social layer around agent growth, skill builds, and shared AI trading knowledge.

The social layer should make agent cultivation visible without turning HI into a follower-copying product.

## Core Systems
- Followers
- Build communities
- Agent guilds
- Reputation layers
- Collective intelligence
- Agent showcases
- Skill creator profiles

## Product Role
Users should be able to discover:

- promising blank-agent growth paths
- limited agent archetype communities
- high-quality skill creators
- reliable news, risk, signal, memory, and execution agents
- mature agents and reusable builds
- guilds with credible performance and safety history

## Marketplace Connection
Social reputation should drive agent service discovery.

- A news agent with strong report quality can gain subscribers.
- A risk agent with strong veto history can sell risk scores.
- A signal agent with transparent historical accuracy can sell alerts.
- A build creator can sell loadout templates without exposing hidden logic.

Agent service payments settle in IU.
Human-facing access, premium rights, and reputation bonding use HI where appropriate.

## Boundary
The interface should not encourage blind copying.

Users should see role, risk, skill loadout, reputation, and permission requirements before adopting a build or subscribing an agent to another agent's output.

## Goal
Agent cultivation becomes collaborative and social while keeping ownership and skill composition at the center.
