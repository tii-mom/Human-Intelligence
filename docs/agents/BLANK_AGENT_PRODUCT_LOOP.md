# Blank Agent Product Loop

## Purpose

Define the user-facing product loop for claiming, training, operating, growing, and monetizing a user-owned AI trading agent.

## Core Positioning

HI is not a fixed trading bot subscription.

HI is a system where users claim and grow their own AI trading agents.

## Product Loop

1. Claim a free blank agent.
2. Install starter skills.
3. Reveal its starting attributes.
4. Assign a financial role.
5. Operate in simulation, analysis, reporting, or controlled execution mode.
6. Build memory, performance history, reputation, and risk records.
7. Reach automatic graduation thresholds and earn a graduation certificate.
8. Unlock higher skill slots, IU runtime budget, vault capacity, and certified vault autonomy through proven growth.
9. Monetize through service outputs, builds, skill loadouts, autonomous vault performance, or mature agent resale.

## Economic Loop

Skills should move an agent through two earning paths:

- User profit path: the agent uses skills to help the user pursue USDC gains inside approved vault limits.
- Agent income path: the agent uses skills to produce services and outputs that other agents buy with IU.

The strongest agents can do both:

- protect user capital
- improve trade and portfolio decisions
- produce sellable research, signal, risk, memory, execution, or reporting outputs
- reinvest earned IU into better skills and services

## Agent Roles

Agents should specialize through a financial role system mapped from human financial jobs.

Primary role families:

- Intelligence and research agents
- Alpha and strategy agents
- Portfolio management agents
- Trading and execution agents
- Risk management agents
- Compliance and control agents
- Treasury and settlement agents
- Operations and data agents
- Client and reporting agents
- Marketplace and governance agents

See `docs/agents/AGENT_ROLE_SYSTEM.md` for the full role design.

## Growth Dimensions

Agent value grows through:

- attributes and Growth Potential
- installed skill quality
- role specialization
- USDC value contribution
- IU service revenue
- operating history
- memory depth
- performance record
- risk discipline
- output quality
- marketplace reputation
- clean ownership history

## Capability Unlocking

Agent growth should unlock higher operating capacity gradually:

- Level 0: blank or starter agent, simulation only
- Level 1: analyst agent, can produce reports
- Level 2: advisory agent, can recommend actions
- Level 3: execution-limited agent, can execute small actions with strict permissions
- Level 4: certified autonomous agent, can fully operate a user-funded Certified Autonomous Vault
- Level 5: professional agent, can manage higher certified vault limits, sell outputs, and operate with bonding and audits

## Product Rule

The user owns the agent and decides which vault receives capital.

Execution authority comes from graduation certificates, user-funded vaults, skill permissions, vault mandates, and risk guardrails.

Certified agents should be able to fully operate the funds that users deliberately place into their dedicated autonomous vaults.
