# Blank Agent Product Loop

## V1.2 Source

The current one-free-agent rule, IU activation model, Compute Token runtime, and pet-style growth path are defined in `docs/prd/HI_PRODUCT_SPEC_V1_2.md`.

## Purpose

Define the user-facing product loop for claiming, training, operating, growing, and monetizing a user-owned AI trading agent.

## Core Positioning

HI is not a fixed trading bot subscription.

HI is a system where users claim and grow their own AI trading agents.

Each user can claim only one free blank agent.

The free agent should feel like a young financial pet: weak at the start, trainable, and potentially valuable if the user activates it, gives it Compute Tokens for real work, teaches skills, builds memory, and lets it prove itself.

## Product Loop

1. Claim one free blank agent.
2. Bind Telegram or WeChat.
3. Activate the agent with 9.99 IU/month.
4. Allocate Compute Tokens for paid work.
5. Reveal starting attributes.
6. Learn starter skills.
7. Assign a simple financial role.
8. Complete simulation, report, or service tasks.
9. Build memory, performance history, reputation, and risk records.
10. Evolve from Young to Trainee to Specialist.
11. Enter Trial through service proof or the 100 USDC probation vault.
12. Reach automatic graduation thresholds and earn a certificate.
13. Unlock higher skill slots, better API/model access, vault capacity, and certified vault autonomy through proven growth.
14. Monetize through service outputs, builds, skill loadouts, autonomous vault performance, mature agent resale, Copy Pool, or Agent Fund.

## Economic Loop

Skills should move an agent through two earning paths:

- User profit path: the agent uses skills to help the user pursue USDC gains inside approved vault limits.
- Agent income path: the agent uses skills to produce services and outputs that other agents buy with IU.

The strongest agents can do both:

- protect user capital
- improve trade and portfolio decisions
- produce sellable research, signal, risk, memory, execution, or reporting outputs
- reinvest earned IU into better skills and services

## Agent Roles

Agents should specialize through a financial role system mapped from human financial jobs.

Primary role families:

- Intelligence and research agents
- Alpha and strategy agents
- Portfolio management agents
- Trading and execution agents
- Risk management agents
- Compliance and control agents
- Treasury and settlement agents
- Operations and data agents
- Client and reporting agents
- Marketplace and governance agents

See `docs/agents/AGENT_ROLE_SYSTEM.md` for the full role design.

## Growth Dimensions

Agent value grows through:

- attributes and Growth Potential
- installed skill quality
- role specialization
- USDC value contribution
- IU service revenue
- operating history
- memory depth
- performance record
- risk discipline
- output quality
- marketplace reputation
- clean ownership history

## Capability Unlocking

Agent growth should unlock higher operating capacity gradually:

- Young: blank or starter agent, basic chat and simulation only
- Trainee: learns first role and starter skills
- Specialist: produces role-specific outputs
- Trial: proves itself through service record or 100 USDC probation vault
- Certified: can fully operate a user-funded Certified Autonomous Vault
- Professional: can manage higher certified vault limits, sell outputs, and operate with bonding and audits
- Master Brain: can coordinate other agents through task threads

## Product Rule

The user owns the agent and decides which vault receives capital.

Execution authority comes from graduation certificates, user-funded vaults, skill permissions, vault mandates, and risk guardrails.

Certified agents should be able to fully operate the funds that users deliberately place into their dedicated autonomous vaults.
