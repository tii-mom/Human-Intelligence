# Blank Agent Ownership

## Purpose
Define the free blank agent model and the ownership rules around user-grown agents.

## Blank Agent
A blank agent is the user's starting AI trading entity.

It is:
- free to claim
- unique
- user-owned
- transferable
- initially limited
- activated through starter skills and installed skills
- eligible for role specialization

## Ownership
The user owns the agent identity, profile, memory history, growth record, reputation record, and transfer history.

Owning an agent does not automatically mean owning every skill installed on it. Skill rights depend on the license type.

## Agent Value
An agent can become more valuable through:
- rare or high-quality skill loadouts
- useful service outputs
- stable operating history
- strong risk discipline
- verified performance records
- useful memory history
- recognizable personality
- role specialization
- clean ownership history

## Transfer and Resale
Users may sell or transfer eligible agents.

When an agent is transferred, the following should remain with the agent:
- agent id
- profile metadata
- level and growth history
- public reputation
- public performance summaries
- public risk record
- ownership history

The following depend on license rules:
- active skill subscriptions
- bought-out skill licenses
- service output rights
- private memories
- exchange permissions
- wallet permissions

## Privacy
Private user data must not transfer with an agent unless the user explicitly exports or authorizes it.

## Product Goal
The blank agent should feel like an ownable AI companion that becomes more capable through skill composition and user guidance.

## Vault Rule
Owning a blank agent does not grant the agent direct control over the user's primary wallet.

After the agent graduates and the user funds a dedicated Certified Autonomous Vault, the agent can fully operate that vault according to its certificate scope.
