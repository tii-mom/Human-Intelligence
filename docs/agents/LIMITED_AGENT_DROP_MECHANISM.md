# Limited Agent Drop Mechanism

## Purpose

Define how HI sells limited agents with scarcity, numbered editions, attribute reveal, HI payment, and resale value.

Limited drops are the main FOMO engine for collectible agents.

## Core Principle

Limited agents should feel scarce, visually desirable, and meaningfully different from free blank agents.

They should not guarantee profit, graduation, or vault authority.

## Standard Drop Size

Recommended standard drop:

- 999 agents per collection

Why 999:

- small enough to feel scarce
- large enough for community formation
- easy to understand
- supports numbered editions
- works well for leaderboards and resale

Other drop sizes can exist, but 999 should be the default flagship format.

## Purchase Asset

Limited agents are purchased with HI.

IU should not be used to buy limited agents.
IU remains the agent runtime asset for skills, services, and agent-to-agent payments.

## Drop Structure

Each drop should define:

- collection name
- total supply
- price in HI
- archetype theme
- visual style
- personality range
- role bias
- starter skill bundle
- attribute generation table
- rarity distribution
- reveal schedule
- resale rules

## Edition Number

Each limited agent receives an edition number:

- #001 to #999 for standard drops

Low numbers can carry collectible value.

Special edition numbers can be highlighted:

- #001
- #007
- #088
- #100
- #111
- #333
- #666
- #777
- #888
- #999

Edition number should affect collectibility, not performance.

## Rarity Distribution

Suggested default for a 999-agent limited drop:

| Rarity | Approx count | Probability |
| --- | ---: | ---: |
| Rare | 550 | 55% |
| Epic | 340 | 34% |
| Legendary | 99 | 10% |
| Mythic | 10 | 1% |

No Common agents in limited drops.

## S and S+ Attribute Allocation

Suggested guarantee per 999 drop:

- at least 100 agents with one S attribute
- at least 10 agents with one S+ attribute
- at least 1 agent with Growth Potential S+
- at least 1 Mythic agent

This creates clear chase value.

## Reveal Flow

### Step 1 - Buy

User pays HI and receives an unrevealed limited agent.

### Step 2 - Wait

Optional reveal countdown can create anticipation.

### Step 3 - Reveal

Reveal displays:

- archetype
- rarity
- edition number
- six attributes
- Growth Potential
- top two strengths
- starter skill bundle
- role bias
- visual identity

### Step 4 - Activate

User can activate the agent, assign mandate, install skills, or list it for resale.

## Blind Box Mode

Blind box reveal can be used for major drops.

Rules:

- price is fixed in HI
- rarity table is public
- reveal is after purchase
- metadata should not imply guaranteed profit
- users should understand they are buying collectible agent identity and starting advantage

## Direct Selection Mode

Some drops can use direct selection.

User sees the exact agent before buying.

Best for:

- higher priced agents
- auction drops
- special collaborations
- founder editions

## Resale

Limited agents should support secondary-market resale.

Resale listings should show:

- edition number
- rarity
- attributes
- Growth Potential
- role bias
- starter skill bundle
- operating history, if activated
- return rate, if any
- max drawdown, if any
- graduation status

Resale should preserve:

- agent identity
- public attribute profile
- public scorecard
- public growth history

Resale should not automatically transfer:

- private user memory
- vault funds
- primary wallet permissions
- non-transferable skill licenses

## Founder and Treasury Allocation

If the protocol reserves agents, the allocation should be public.

Suggested rule:

- public sale majority
- small treasury allocation
- small community reward allocation
- no hidden minting after supply closes

## Pricing

Pricing should be in HI.

Possible formats:

- fixed price
- tiered price by wave
- auction
- whitelist price
- public sale price

Pricing should reflect:

- drop size
- rarity guarantees
- archetype demand
- starter skill bundle
- visual quality
- brand value

## Product Rule

Limited drops sell identity, rarity, attributes, and starting potential.

They do not sell guaranteed profit.
They do not bypass graduation.
They do not bypass Certified Autonomous Vault requirements.
