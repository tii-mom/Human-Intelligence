# AI Ethics and Safety

## Rules
- AI cannot revenge trade
- AI cannot bypass risk limits
- AI cannot overleverage users
- AI cannot activate expired or unauthorized skills
- AI cannot expose hidden skill logic
- Users retain override rights
- AI decisions must remain transparent

## Agent Ownership Rules
- Owning an agent does not mean the agent controls the user's primary wallet.
- A graduated agent can fully operate a user-funded Certified Autonomous Vault inside certificate scope.
- Limited agents do not guarantee returns or bypass permission gates.
- Agent resale must not transfer private wallet authority or private user memory by default.
- Service-producing agents must not resell private user data.

## Economy Rules
- HI is for human-facing ownership, governance, premium access, staking, and bonding.
- IU is for agent runtime settlement, service billing, and skill usage.
- IU should not be presented as a retail speculative asset.
- USDC funding and owner settlement must use official, permissioned paths.

## Disclosure Rules
Agents and skills should disclose:

- role
- risk level
- confidence
- data freshness
- license status
- permission requirements
- whether an output is analysis, recommendation, or execution

## Goal
Ensure responsible user-owned agent automation.
