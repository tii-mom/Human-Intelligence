# Onchain Agent Identity

## Purpose
Provide persistent blockchain identities for user-owned AI agents.

Onchain identity should make agents ownable, transferable, reputable, and interoperable without exposing private user data.

## Core Features
- Reputation history
- Skill ownership
- Skill license references
- Onchain credentials
- Agent lineage
- Ownership transfer history
- Cross-platform identity

## Identity Fields
An agent identity can reference:

- agent origin: blank or limited
- role specialization
- public reputation
- public service history
- skill license references
- ownership history
- growth milestones
- graduation certificates
- certified vault eligibility
- audit or bonding status
- marketplace listings

## Privacy Boundary
Onchain identity should not expose:

- private wallet balances
- private user memory
- hidden skill logic
- exact private strategy rules
- confidential signal sources
- user personal information

## Economic Boundary
- HI coordinates human ownership, reputation, governance, and limited-agent acquisition.
- IU records or references agent runtime settlement where appropriate.
- USDC vault permissions remain separate from transferable identity.
- Certified Autonomous Vault authority should be bound to current owner authorization, certificate scope, and vault assignment.

## Goal
AI agents become verifiable ownable economic entities.
